Frequently Asked Questions

What is a bounce checker?

In TF2 rocket jumping, certain physics bugs allow players to touch teleport triggers without being warped away. While players often trigger these by accident, known as "random bounces", it is possible to execute them consistently using Low Input Strategies (LIS). A bounce checker is a tool for finding consistent, executable LIS "setups."

How does this bounce checker work?

The tool works by pairing movement binds (like +forward) with action binds (like +attack). To perform a setup, you hold down the movement bind first, followed by the action bind. For simpler setups, a single tap of the action bind is enough. For more complex setups, such as when holding the action button triggers a continuous pogo, you may need to manually crouch or uncrouch to fine-tune your speed from the explosions.

You can look for specific setups using the "Preferences" tab. If you are a beginner, I highly recommend maxing out the "Simple" preference. If you are an advanced jumper, try increasing the weight of "Automated" setups. These are fancy setups where you can simply hold down M1 to hit the bounce, which is a major feature of this tool.

How was this tool made?

This is a long-standing passion project of mine. By analyzing TF2 source code (open-sourced by Valve themselves), I built a custom, physics-accurate simulation of TF2's movement engine in Python. I then created this bounce checker on top of that simulation, programmatically testing hundred of thousands of LIS combinations to see which ones produce a bounce. Every setup on this website was generated by that Python tool. While the website currently displays pre-calculated results rather than running the engine live, I hope to port the simulation to JavaScript in the future to allow on-demand generation.
